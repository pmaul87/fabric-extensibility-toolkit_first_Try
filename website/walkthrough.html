Walkthrough

Product flow

This page follows the README walkthrough and keeps space for the screenshots you plan to add later.

Step 1

Set up Lakehouse and notebooks

  • Select or create a Lakehouse.
  • Select or create an Environment and configure dependencies.
  • Deploy the extraction notebooks.
  • Choose workspaces to extract and run the job.

Step 2

Open the lineage explorer and select a node

  • Open the explorer panel in the Lineage view.
  • Select a dataset, table, column, report, or visual node.
  • Use the selected node as the context for deeper analysis.

Step 3

Inspect related nodes in detail and graph views

  • Review upstream and downstream dependencies in the detail panel.
  • See the connected nodes highlighted inside the graph.
  • Adjust depth and direction controls to refine the view.

Step 4

Show query steps for columns

  • Open the transformation or query lineage section for a selected column.
  • Inspect the steps that produced or transformed the field.
  • Use this information before making semantic model changes.

Step 5

Manage requirement tickets

  • Create and edit tickets inside the integrated board.
  • Link tickets to lineage nodes for impact-aware planning.
  • Track progress with Kanban columns from Backlog to Done.
